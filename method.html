Method Overloading

  • In Java it is possible to define two or more methods within the same class that share the same name, as long as their parameter declarations are different. When this is the case, the methods are said to be overloaded, and the process is referred to as method overloading.
  • Method overloading is one of the ways that Java supports polymorphism.
  • When an overloaded method is invoked, Java uses the type and/or number of arguments as its guide to determine which version of the overloaded method to actually call.
  • Thus, overloaded methods must differ in the type and/or number of their parameters.
  • While overloaded methods may have different return types, the return type alone is insufficient to distinguish two versions of a method. 

		
class overloading
{
    void test()
    {
        int a=10;
        System.out.println(a);
    }
    void test(int a)
    {
       System.out.println("a is: "+a);
    }
    void test(int a, int b)
    {
        System.out.println("a and b: "+a+" "+b);
    }
    double test(double a)
    {   
        System.out.println("a is: "+a);
        return a*a;
    }
}
class example
{
    public static void main(String args[])
    {   
        double result;
        overloading object= new overloading();
        object.test();
        object.test(10);
        object.test(10,20);
        result=object.test(13.2);
        System.out.println("Square is:"+result);
    }
}

OUTPUT:

		
10
a is: 10
a and b: 10 20
a is: 13.2
Square is:174.23999999999998

When an overloaded method is called, Java looks for a match between the arguments used to call the method and the method’s parameters. However, this match need not always be exact. In some cases, Java’s automatic type conversions can play a role in overload resolution. 

		
class overloading
{
    void test()
    {
        int a=10;
        System.out.println(a);
    }
    void test(int a, int b)
    {
        System.out.println("a and b: "+a+" "+b);
    }
    void test(double a)
    {
        System.out.println(a);
    }
}
class example
{
    public static void main(String args[])
    {   
        overloading object= new overloading();
        int a=88;
        object.test();
        object.test(10,20);
        object.test(88);
        object.test(13.2);
    }
}

OUTPUT:

		
10
a and b: 10 20
88.0
13.2

Overloading Constructors:

In addition to overloading normal methods, you can also overload constructor methods. In fact, for most real-world classes that you create, overloaded constructors will be the norm, not the exception. 

		
class box
{   
    int height;
    int width;
    int length;
    box()
    {
        height=width=length=1;
    }
    box(int h, int w, int l)
    {
        height=h;
        width=w;
        length=l;
    }
    box(int a)
    {
        height=width=length=a;
    }
    int volume()
    {
        return height*width*length;
    }
}
class example
{   
    public static void main(String args[])
    {
        box box1= new box();
        box box2= new box(10,20,30);
        box box3= new box(10);
        int a= box1.volume();
        System.out.println("Volume is: "+a);
        a= box2.volume();
        System.out.println("Volume is: "+a);
        a= box3.volume();
        System.out.println("Volume is: "+a);
    }
}

OUTPUT:

		
Volume is: 1
Volume is: 6000
Volume is: 1000

The proper overloaded constructor is called based upon the parameters specified when new is executed.
